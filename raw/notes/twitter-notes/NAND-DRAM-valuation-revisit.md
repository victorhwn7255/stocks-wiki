# 存储即智力：AI时代NAND/DRAM的价值重估

我认为，市场严重、严重低估了NAND和DRAM最终的价格和需求量。

我不敢相信需要由我来指出这一点。

第一阶段：训练需求

DRAM被认为是模型训练阶段容纳参数的关键。随着模型越来越大，更多的HBM被需要，用于桥接模型训练。那是2024到2025年的故事。

第二阶段：推理需求

HBM、DRAM乃至NAND被需要用于大规模部署模型，数以百万计的部署实例，触发了第二波短缺。这就是我们正在目睹的阶段。

大多数人会认为存储需求到此为顶，然后逐步下降。

不。

第三阶段：运营优化

有没有想过为什么DeepSeek用起来这么便宜？

他们大规模、重度缓存，使得缓存读取为用户节省了50%到80%的token。（DeepSeek背后的幻方量化可能在早年为其对冲基金业务以低价采购了大量存储。）

DeepSeek V4 Flash的定价已经把这一点写在了明面上：缓存命中的token价格比缓存未命中低98%。这不是营销策略，这是他们的成本结构在说话。缓存读取的真实成本确实比GPU计算低98%。

这将是未来的核心分水岭。 用NAND和DRAM缓存，远比用GPU重新计算便宜。而全世界的人每天都在问非常相似的问题。因此，每多部署一TB的NAND，回报率极高，为AI云节省的成本极大。

简单算一笔账

一个问题如果会被问5次，缓存节省80%的计算成本。 被问10次？节省90%。 被问100次？节省99%。

NAND读取一旦部署，边际成本基本为零。而每一次GPU计算都意味着电力、散热，成本叠成本。这不是工程上的近似，这是热力学：计算有不可消除的能量下限，存储读取趋近于零。

一个问题可能需要0.05美元的算力来回答。被重复问10次就是0.50美元。将其缓存在NAND上的成本大约0.01美元（很可能更低）。这是5,000%的ROI。

以目前H200约3美元/小时的租赁价格反算：NAND价格需要涨到约2.80美元/GB，缓存才会变得不如重新计算经济。（这个数字是Opus帮我反算的。）

当前NAND价格是0.10到0.20美元/GB。天花板在头顶10到28倍的位置。 市场把NAND当作大宗商品来定价，但它头上悬着一把10倍的定价保护伞，没有人看到。

而且模型在持续膨胀。两年前前沿模型是70B参数，今天是1到5T。模型越大，单次查询的GPU成本越高，NAND的打平价天花板就被撑得越高。AI产业的发展方向本身在持续强化这把保护伞，不是削弱它。

想想AI时代的Google

把Google搬到AI时代来想。需要被缓存的不再是URL和网页。而是所有人类将会提问的知识和问题本身，附带语义近似索引和压缩。

根据搜索引擎的使用数据推算，大约50%的问题会被重复提问5次以上。这意味着：

大约50%的AI查询，用缓存来服务比用GPU重新计算更经济。长期来看，这将削减50%的GPU需求，同时大规模拉升NAND需求。

而剩下的50%的问题，可以被拆解为「原子级」子问题，而这些子问题本身又是可以缓存的。拆解本身的成本只是一次完整推理的零头，净经济账依然压倒性地有利。

并发读取：需求的非线性放大器

这里还有一个被忽略的维度。

热门问题不是存一份缓存就够了。「Python怎么写for循环」每秒可能被全球几千人同时问。一块SSD一次只能服务一个读取请求。如果每秒有5000个并发请求命中同一条缓存，你需要这条缓存存在至少5000个可以并行读取的物理位置上。

这和Netflix是同一个逻辑：最火的剧不是存一份，是在全球几千个边缘节点各存一份，因为瓶颈永远是并发读取带宽，不是存储容量。

最有价值的缓存条目，恰恰是被复制最多份的。NAND需求和查询频率之间是乘法关系：一次是因为值得缓存，一次是因为并发读取需要的副本数。

竞争越多，缓存需求越大

短期来看，模型供应商越多，NAND总需求越大。因为OpenAI的缓存不能给Anthropic用，DeepSeek的缓存不能给Google用。同样的问题被缓存在几十个独立系统里。供应商数量本身就是NAND需求的乘数。

但长期来看，缓存的规模经济会碾压一切。

用户多 → 查询分布覆盖广 → 缓存命中率高 → 成本低 → 可以降价 → 用户更多。

这是一个残酷的正反馈循环。一家服务10亿用户的公司，80%的查询命中缓存。一家服务100万用户的公司，可能只有30%命中。两者的单次查询成本差距不是2倍3倍，而是一个数量级。缓存做得好的，会以十分之一的成本提供好得多的结果，碾压所有小对手。

终局看起来和搜索引擎一样：三到五家全球级运营商，各自坐拥EB级的缓存智力池，任何试图从零开始靠GPU硬算竞争的新进入者，在成本结构上不可能匹配。

这是AI serving经济学的终极答案：怎么让服务成本不随用户数量线性增长。 缓存打破线性。OpenAI九亿周活用户，每多一个查询就多一份GPU成本，毛利率被钉死在30%到40%，所以在250亿美元年收入下还在亏损。这不是经营问题，是数学问题。DeepSeek是第一个认真在解这道数学题的。

Google解决过这个问题：10亿人搜同一个关键词，只计算一次排序，然后服务缓存，毛利率55%以上。Netflix解决过这个问题：一亿人看同一部剧，只存储一份在边缘节点。AI serving正在第三次解同一道题。

而且这一次收敛会更快。Google靠爬虫慢慢爬索引用了10年，Netflix靠物理节点慢慢铺用了8年。AI缓存池的积累速度等于用户查询的速度，10亿用户每天自动贡献25亿条可缓存的信号，不需要任何额外的采集成本。

更进一步，未来最成功的模型不会是benchmark分数最高的，而是为可缓存性而设计的。推理过程被拆解成标准化的中间步骤，每一步可独立缓存和复用。训练目标函数里可能会加入缓存命中率作为优化指标。这听起来反直觉，但经济激励太强了，一定会发生。

而且，他们甚至不需要把省下来的钱给你

云服务商按token收费。用户不知道也不关心这个token是GPU算出来的还是NAND读出来的。答案质量一样，延迟更低，用户体验甚至更好。每一次缓存命中，服务商收的是GPU计算的价格，付的是NAND读取的成本。

这不是优化。这是一台印钞机。

所有其他服务商会被迫跟进，要么大规模部署NAND做缓存，把cache hit rate也推到70%以上，要么继续每个请求都用GPU算，成本比对手贵50倍，定价上被碾压。这不是战略选择，这是生存选择。

两种智力

作为人类，我们一直知道智力有两种形态：一种是反应快的人，一种是博闻强记的人。

市场已经为「反应快」（GPU）充分定价了。市场仍然严重低估「博闻强记」（NAND/DRAM）的价值。

这个需求有多大？

在AI共享智力的云时代，通过NAND实现的缓存，需要10,000 EB级别的存储。制造商需要数年才能追上这个需求，即使他们全力扩产。

提示：他们不会如你所愿那样快地扩产。因为他们的员工现在都已经是千万富翁了，正在享受一个月的长假。

而无论最终哪家模型公司赢得AI之战，NAND供应商赢得的是战后的和平红利。
